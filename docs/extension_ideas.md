# 还能加点啥

基础功能都齐了，想着还有没有能锦上添花的。随手记了几个，大家看看哪个有意思。

## 主机风险评分

现在规则挺多的，告警一条条往外蹦，但缺一个先看谁的感觉。实际的安全产品（SIEM、XDR）都有主机维度的综合风险分——不是只看某条告警的等级，而是把同一台机器的告警等级分布、它在攻击链里走到了哪个阶段、这台机器的资产重要性、它偏离基线的程度这些维度合在一起算一个分。

我们其实已经有零散的部件了。告警等级在 alert 表里存着，攻击链阶段 attack_chain 里已经按 src_ip 做了分组和阶段排序，资产重要性在 false_positive 的 asset_importance 字典里配，基线偏离在 baseline_deviation 和 bandwidth_spike 这些规则里已经实时在比。把这些串一下就行，不用从零造轮子。

大致做法：给每个 src_ip 维度的维度加权打分——告警等级（CRITICAL 权重高）、攻击链是否完整（scan 到 lateral_movement 几步都走到的加分）、是否命中高价值资产、基线偏离幅度。仪表盘上多一个高危主机 Top N 的卡片，点进去能看到这个主机为什么分数高（拆到各维度得分）。dashboard_page 已经有 AttackChainAnalyzer 和 repositories，数据源都在。

## 演示数据集 + 一键演示

功能写得差不多了，但怎么给老师看是个问题。总不能现场找个 pcap，结果跑出来啥也没有。想弄一两组合成的 pcap，用 Scapy 构造几条预设的攻击链路——比如从主机扫描到 SQL 注入到反弹 shell 到 C2 外联的完整链路。然后在 GUI 的流量监控页面或仪表盘上加一个加载演示数据的按钮，点一下就自动导入、跑检测、刷新告警中心和攻击链视图。

好处是老师验收的时候点个按钮就能看到完整流程，不用自己找数据，也不用担心网络问题或者随机流量什么都检测不出来。我们自己测试的时候也能反复用，相当于一个可重复的验收用例。

实现的话就是写一个 generate_demo_pcap.py 脚本，用 Scapy 的 IP/TCP/Raw 层构造包含攻击特征的数据包——比如在 Raw payload 里塞 SQL 注入语句、XSS payload、反弹 shell 命令——然后写成 pcap 文件放到 sample_data 里。GUI 那边加个按钮调用已有的 pcap 导入流程，路径写死就行。

## 告警趋势分析

仪表盘现在展示的是当前快照，总数、分布、Top IP、Top 端口这些。但实际运维场景里比较在意时间维度——今天告警量跟昨天比算正常吗、最近一小时有没有突然的进攻潮、某条规则是不是越来越吵。

用 SQLite 里 alerts 表的时间戳做聚合，按小时或者按天分桶，画出告警量的时序折线图。再加一个简单的异常检测——如果当前小时的告警量超过了历史均值+2倍标准差，标红提示。数据都在 SQLite 里，AlertRepository 已经有 count_by_severity 和 count_by_type，加一个按时间段分组的查询就行。前端 dashboard_page 里加个时序图区域，ChartWidget 已经封装好了，配数据就行。

## 规则效果反馈（备选）

告警中心里确认和忽略的按钮现在只是改了个状态，点了就点了。但如果把用户的操作行为记下来，回过头看哪些规则老是被忽略、哪些规则阈值调过之后确认率上去了，其实对运维挺有用的。

做法就是在规则管理页面的表格里加一列，显示这条规则最近产生了多少告警、其中被确认和被忽略的比例。数据来源就是 alerts 表里 status 字段的分布（unconfirmed / confirmed / ignored），按 rule_id 聚合就行。如果某条规则被忽略的告警占比超过一半，在表格里标个黄底提醒一下。不算复杂，但让规则管理从静态配置变成了有反馈的闭环。
